## Regole Git Commit e Push

**NON eseguire MAI `git commit` o `git push` senza prima chiedere esplicita conferma all'utente.**

1. Quando il lavoro è pronto, mostra all'utente un riepilogo delle modifiche (`git status` / `git diff --stat`)
2. Chiedi conferma prima di procedere con il commit
3. Chiedi conferma prima di procedere con il push
4. Esegui commit/push SOLO quando l'utente lo chiede esplicitamente
